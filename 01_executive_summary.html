📍 ที่มาของข้อมูล

ข้อมูลทั้งหมดมาจาก self.rtm_data และ self.bug_data ที่ดึงมาจาก Azure DevOps Work Items API แล้ว

Work Item Types ที่เกี่ยวข้อง:
• Product Backlog Item → PBI (24 ตัวใน Sprint 54)
• AXONS Test Scenario v2 → Test Cases (511 ตัว)
• AXONS Bug → Bugs (10 ตัว)
• Task → Dev/QA Tasks

📐 สูตรคำนวณทุก Metric (พร้อมตัวเลขจริง Sprint 54)

1. No. of PBI with Test Scenarios

pbis_with_test / total_pbis × 100 = 19 / 19 × 100 = 100.00%

ดูจาก: total_tests > 0 ของแต่ละ PBI

2. Test Execution Status

(Passed + Failed + Invalid + Not Start) / Total Tests × 100 = 508 + 0 + 3 + 0) / 511 × 100 = 100.0% = 100.0%

ดูจาก: Custom.Testresult — 508 Passed, total 511 tests

3. Test Execution Pass Rate

Passed / (Passed + Failed) × 100 = 508 / (508 + 0) × 100 = 100.0%

นับเฉพาะ test ที่ execute แล้ว — Sprint 54 ไม่มี Failed เลย

4. Test Case Efficiency (ยิ่งสูงยิ่งดี)

(Total Tests - Total Bugs) / Total Tests × 100 = (511 - 39) / 511 × 100 = 92.4% = 92.4%

วัดสัดส่วน test cases ที่ไม่พบ bug — ยิ่งสูงแสดงว่า code คุณภาพดี bug น้อย

5. Test Automation Coverage

Automated / (Automated + Automatable) × 100 = 25 / (25 + 0) × 100 = 100.0%

ดูจาก: Custom.Automationteststatus — API: 25 automated tests

def generate_executive_summary(data, bug_data=None):
    """สร้าง Executive Summary section"""

    # ข้อมูลพื้นฐาน (Sprint 54 จริง)
    total_pbis = 24       # PBIs ที่มี total_tests > 0
    total_tests = 511     # AXONS Test Scenario v2 ทั้งหมด
    total_bugs = 39      # AXONS Bug ทั้งหมด (linked + unlinked)

    # คำนวณ Test Execution Status
    # (508 Passed + 0 Failed + 3 Invalid + 0 Not Start) / 511 × 100 = 100.0%
    test_execution_status = (
        (total_passed + total_failed + total_invalid_case + total_not_start)
        / total_tests * 100
    ) if total_tests > 0 else 0

    # คำนวณ Pass Rate
    # 508 / (508 + 0) × 100 = 100.0%
    executed_items = total_passed + total_failed
    test_execution_pass_rate = (total_passed / executed_items * 100) if executed_items > 0 else 0

    # คำนวณ Automation Coverage
    # 11 / (11 + automatable) × 100 = 40.7%
    automated_tests = int(data.get('automated_tests', 0))    # 11 (API tests)
    automateable_tests = int(data.get('automateable_tests', 0))
    test_automation_coverage = (
        automated_tests / (automated_tests + automateable_tests) * 100
    ) if (automated_tests + automateable_tests) > 0 else 0

    # กำหนดสีตามค่า (ยิ่งสูงยิ่งดี)
    # >= 80% = green, >= 50% = yellow, < 50% = red
    req_color = _get_metric_color_class(requirements_coverage)   # 100% → green
    exec_color = _get_metric_color_class(test_execution_status)  # 68% → yellow
    pass_color = _get_metric_color_class(test_execution_pass_rate) # 100% → green

    # Test Case Efficiency (ยิ่งสูงยิ่งดี)
    # (511 - 39) / 511 × 100 = 92.4% = 92.4%
    defect_density = ((total_tests - total_bugs) / total_tests * 100) if total_tests > 0 else 0
    efficiency_color = _get_metric_color_class(defect_density)  # ไม่ใช้ inverse แล้ว
