📐 นิยามสถานะ Pipeline

result (API)แสดงเป็นIconสีความหมาย
succeeded Passed ✅ #10b981 เขียว ทุก test ผ่าน
failed Failed ❌ #ef4444 แดง มี test ที่ไม่ผ่าน + แสดง error box สีแดง
partiallySucceeded Partial ⚠️ #f59e0b เหลือง ผ่านบางส่วน + แสดง error box สีแดง
inProgress (state) Running 🔄 #3b82f6 น้ำเงิน กำลังรันอยู่
canceled Canceled ⛔ #6b7280 เทา ถูกยกเลิก
💡 ลำดับการตรวจสอบ: ระบบตรวจสอบ state ก่อน แล้วค่อยดู result
ถ้า state = inProgress → แสดง Running (ไม่ดู result)
ถ้า result = failed/partiallySucceeded → แสดง error box สีแดงด้านล่าง card

📍 ที่มาของข้อมูล (3 ขั้นตอน)

ขั้นที่ 1 ดึงรายชื่อ Pipeline ทั้งหมดในโปรเจค 
GET https://dev.azure.com/{org}/{project}/_apis/pipelines?api-version=7.0
ขั้นที่ 2 ดึง Build ล่าสุดของแต่ละ Pipeline (ใช้ Build API) 
GET https://dev.azure.com/{org}/{project}/_apis/build/builds?definitions={pipeline_id}&$top=5&api-version=7.0
เลือก build ที่ status = completed ล่าสุด
ขั้นที่ 3 ถ้า Failed → ดึง Failure Details จาก Timeline API 
GET https://dev.azure.com/{org}/{project}/_apis/pipelines/{id}/runs/{run_id}/timeline?api-version=7.0-preview.1
ดึง: failed job name, error message, HTTP error codes
