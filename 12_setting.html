⚙️ Setting
การติดตั้งและปรับใช้ QA Test Report Generator กับทีมของคุณ

ติดตั้งครั้งเดียว  ·  ใช้ Amazon Q ตั้งค่าให้อัตโนมัติ  ·  ใช้งานได้ทุก Sprint โดยไม่ต้องแก้โค้ดใดๆ

📦 สิ่งที่ต้องเตรียมก่อนเริ่ม

รายการดาวน์โหลดที่หมายเหตุ
✅ Python 3.xpython.org/downloadsติดตั้งครั้งเดียว
✅ Gitgit-scm.comติดตั้งครั้งเดียว
✅ VS Codecode.visualstudio.comติดตั้งครั้งเดียว
✅ Amazon Q Extensionติดตั้งใน VS Code (STEP 3)ติดตั้งครั้งเดียว
✅ PAT Tokenสร้างใน Azure DevOps (STEP 6)ต่ออายุทุก 90 วัน
💡 ถ้ามีรายการไหนแล้ว ข้ามไปได้เลย ไม่ต้องติดตั้งซ้ำ

STEP 1  ·  ติดตั้ง Python ข้ามได้ถ้ามีแล้ว

ตรวจสอบว่ามีหรือยัง

เปิด Command Prompt (กด Win+R → พิมพ์ cmd → Enter) แล้วพิมพ์:

python --version
✅ เห็น Python 3.x.x  →  มีแล้ว ข้ามไป STEP 2 ได้เลย
❌ เห็น 'python' is not recognized  →  ต้องติดตั้ง ทำตามขั้นตอนด้านล่าง

ขั้นตอนติดตั้ง

  1. เปิดเว็บ python.org/downloads
  2. คลิกปุ่ม Download Python 3.x.x (ปุ่มเหลืองใหญ่)
  3. เปิดไฟล์ที่ดาวน์โหลดมา
  4. ⚠️ ติ๊ก ☑ "Add Python to PATH" ก่อนกด Install ← สำคัญมาก!
  5. คลิก Install Now → รอติดตั้งเสร็จ
  6. ปิด cmd แล้วเปิดใหม่ → พิมพ์ python --version อีกครั้ง
✅ เห็น Python 3.x.x  →  ติดตั้งสำเร็จ ไป STEP 2

STEP 2  ·  ติดตั้ง Git ข้ามได้ถ้ามีแล้ว

ตรวจสอบว่ามีหรือยัง

git --version
✅ เห็น git version x.x.x  →  มีแล้ว ข้ามไป STEP 3 ได้เลย
❌ เห็น 'git' is not recognized  →  ต้องติดตั้ง

ขั้นตอนติดตั้ง

  1. เปิดเว็บ git-scm.com/download/win → ดาวน์โหลด
  2. เปิดไฟล์ติดตั้ง → กด Next ไปเรื่อยๆ
  3. เมื่อเจอหน้า "Adjusting your PATH environment"
  4. ⚠️ เลือก "Git from the command line and also from 3rd-party software" ← สำคัญมาก!
  5. กด Next จนจบ → Install → ปิด cmd แล้วเปิดใหม่
✅ เห็น git version x.x.x  →  ติดตั้งสำเร็จ ไป STEP 3

STEP 3  ·  ติดตั้ง VS Code และ Amazon Q ข้ามได้ถ้ามีแล้ว

ติดตั้ง VS Code

🧰 Windows
  1. เปิดเว็บ code.visualstudio.com
  2. คลิก Download for Windows
  3. เปิดไฟล์ → กด Next ไปจนจบ → Install
🍎 Mac
  1. เปิดเว็บ code.visualstudio.com
  2. คลิก Download for macOS
  3. เปิดไฟล์ .zip → ลาก Visual Studio Code.app ไปที่โฟลเดอร์ Applications
  4. เปิด VS Code จาก Applications

ติดตั้ง Amazon Q Extension (ทำเหมือนกันทั้ง Windows และ Mac)

  1. เปิด VS Code
  2. กด Ctrl + Shift + X (Windows) หรือ Cmd + Shift + X (Mac)
  3. ค้นหา Amazon Q → คลิก Install (ของ AWS)
  4. คลิก Sign In ที่ pop-up → เลือก Use for Free with Builder ID
  5. Login ด้วยอีเมล → Browser เปิดขึ้นมา → กลับมา VS Code → คลิก Open
✅ เห็นไอคอน Q ที่แถบซ้าย VS Code  →  พร้อมใช้งาน ไป STEP 4

STEP 4  ·  Clone Code ลงเครื่อง

เปิด Terminal ใน VS Code ด้วย Ctrl + ` แล้วพิมพ์ทีละบรรทัด:

cd C:\
mkdir QA-Tools
cd QA-Tools
git clone https://dev.azure.com/AXONS-Strategic-And-Digital-Transformation/QA%20Team%20Playground/_git/QA-TESTREPORT
code QA-TESTREPORT
✅ VS Code เปิดโฟลเดอร์ QA-TESTREPORT ขึ้นมาใหม่  →  สำเร็จ ไป STEP 5
❌ Authentication failed  →  ต้องสร้าง PAT ก่อน → ทำ STEP 6 แล้วกลับมาใหม่

STEP 5  ·  ติดตั้ง Python Packages

ใน Terminal ของ VS Code:

cd AXONS_Report
pip install requests python-dotenv
✅ Successfully installed  →  โอเค ไป STEP 6
✅ Requirement already satisfied  →  โอเค มีอยู่แล้ว ไป STEP 6

STEP 6  ·  สร้าง PAT จาก Azure DevOps

🔑 PAT คือรหัสผ่านพิเศษให้ Script ดึงข้อมูลจาก Azure DevOps แทนการ Login

ขั้นตอน

  1. เปิด Azure DevOps ขององค์กร → Login
  2. คลิก รูปโปรไฟล์ มุมขวาบนสุด → User Settings → Personal Access Tokens
  3. คลิก + New Token

กรอกข้อมูลในฟอร์ม

Fieldค่าที่ต้องใส่
NameQA-Report-2026
Expiration90 days
ScopesCustom defined → ติ๊ก ☑ Work Items → Read, ☑ Build → Read, ☑ Code → Read
  1. คลิก Create
  2. ⚠️ Copy Token ทันที! เห็นแค่ครั้งเดียว ถ้าปิดหน้าต่างจะหายไปเลย!
  3. วางใน Notepad เก็บไว้ก่อน

STEP 7  ·  ใช้ Amazon Q ตั้งค่าทีม สำคัญที่สุด!

Amazon Q จะแก้ไฟล์ .env ให้อัตโนมัติ — ไม่ต้องแก้โค้ดเองเลย

ขั้นตอน

  1. คลิกไอคอน Q ที่แถบซ้าย VS Code
  2. คลิก Open Chat
  3. คัดลอก Prompt ด้านล่าง → แก้ข้อมูลใน [ ] ให้ตรงกับทีมของคุณ → วางใน Chat → กด Enter

Prompt ที่ต้องใช้

ช่วยแก้ไฟล์ AXONS_Report/.env ให้หน่อยครับ ด้วยข้อมูลดังนี้:
Organization : [ชื่อ Org เช่น AXONS-FIT-Business-and-CPTG]
PAT : [วาง Token จาก STEP 6]
Project : [ชื่อ Project เช่น AgriTech]
Team Code : [ดูจาก Azure DevOps → Project Settings → Teams]
Email ผู้รับ : [email1@domain.com;email2@domain.com]
กำหนดให้:
- AZURE_DEVOPS_SPRINT ปล่อยว่าง (auto-detect Sprint ปัจจุบัน)
- AZURE_DEVOPS_EMAIL=true
- ไฟล์อื่นๆ ไม่ต้องแตะ

ผลลัพธ์ที่จะเห็นใน Amazon Q

แก้ไขไฟล์ AXONS_Report/.env แล้วครับ:
AZURE_DEVOPS_ORG  = "AXONS-FIT-Business-and-CPTG" ✓
AZURE_DEVOPS_PAT  = "abc123xxxxx..." ✓
AZURE_DEVOPS_PROJECT = "AgriTech" ✓
AZURE_DEVOPS_SPRINT  = "" ← ว่าง = auto-detect ✓
AZURE_DEVOPS_TEAM   = "PK" ✓
AZURE_DEVOPS_RECIPIENT = "email@domain.com" ✓
✅ ไฟล์ .env ถูกบันทึกอัตโนมัติ  →  ไป STEP 8

STEP 8  ·  รัน Script สร้าง Report

ใน Terminal ของ VS Code:

python REPORT_Generator.py

ระหว่างรันจะเห็น:

$ python REPORT_Generator.py
Using config: config.json
🔍 Testing Azure DevOps connection...
🔄 Fetching work items...
📄 Generating HTML report...
📚 Creating wiki page...
📧 Checking email configuration...
✅ Report generation completed successfully!
✅ เห็น "completed successfully!"  →  สำเร็จ ไป STEP 9
❌ เห็น "401 Unauthorized"  →  PAT ผิด กลับไป STEP 6 สร้าง PAT ใหม่
❌ เห็น "ModuleNotFoundError"  →  รัน pip install requests python-dotenv

STEP 9  ·  เปิดดู Report

explorer reports
📂 โฟลเดอร์เปิดขึ้นมา:

└── index.html  ← ดับเบิ้ลคลิกเพื่อเปิด Report ใน Browser

🔁 Sprint ต่อไป — ไม่ต้องทำอะไรเพิ่มเลย

AZURE_DEVOPS_SPRINT="" ปล่อยว่างไว้เสมอ — Script จะ auto-detect Sprint ปัจจุบัน เองทุกครั้ง

แค่รันคำสั่งนี้พอ ✅ 
python REPORT_Generator.py
