📊 คู่มือ QA Test Report Generator
อธิบายทุกหัวข้อในรายงาน ตั้งแต่ที่มาของข้อมูล สูตรคำนวณ ความหมายของสี จนถึงโค้ดที่ใช้สร้าง

คู่มือนี้จัดทำขึ้นเพื่อให้ทุกคนในทีมที่เกี่ยวข้องสามารถอ่านและทำความเข้าใจรายงาน QA Test Report ได้อย่างถูกต้อง ครอบคลุมตั้งแต่ที่มาของข้อมูล สูตรคำนวณ ความหมายของสีและตัวเลขที่แสดงในแต่ละหัวข้อ รวมถึงโค้ดที่ใช้สร้างรายงาน

ภาษาและ Tools ที่ใช้

Tool / ภาษาใช้ทำอะไร
Python 3ภาษาหลักที่ใช้เขียนทั้งหมด
requestsดึงข้อมูลจาก Azure DevOps API
Azure DevOps REST APIแหล่งข้อมูลหลัก (Work Items, Pipelines, Iterations)
HTML + CSSสร้างหน้า Report ที่แสดงผล
JSONรูปแบบข้อมูลที่ API ส่งกลับมา
Power Automateส่ง Email Report

Flow การทำงานทั้งหมด

main() เริ่มทำงาน │ ├── 1. อ่าน config.json + .env → ได้ PAT, Project, Team, Sprint │ ├── 2. auto_detect_current_sprint() → หา Sprint ปัจจุบันจาก Azure DevOps │ ├── 3. fetch_work_items() → ดึง PBI ทั้งหมดจาก Sprint Backlog │ └── process_pbi(pbi_id) สำหรับแต่ละ PBI: │ ├── get_linked_test_cases() → ดึง Test Scenarios │ ├── get_linked_task_details() → ดึง Tasks │ └── get_linked_bugs() → ดึง Bugs │ ├── 4. search_unlinked_bugs_in_sprint() → หา Bug ที่ไม่ได้ link กับ PBI │ ├── 5. save_html_report() → สร้าง HTML Report (11 หัวข้อ) │ ├── 6. create_sprint_wiki() → สร้าง Wiki page ใน Azure DevOps │ └── 7. generate_email_report() → ส่ง Email ผ่าน Power Automate
